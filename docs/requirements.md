# IT補助金申請 受付システム 要件定義（抜粋）

このシステムは、IT導入補助金の申請サポートを LINE 経由で開始する前に、申請者から必要書類と基本情報を収集・事前確認するためのWebシステムです。

## 業務フロー

1. 申請サポートの事前面談・紹介経由で、管理者がシステム内で申請URL（トークン付き）を発行
2. URLを LINE などで申請者に共有
3. 申請者はブラウザで URL を開き、以下を入力・提出
   - 法人/個人事業主の区分
   - 代表者連絡先、LINE表示名
   - GビズID の取得状況（**パスワードは絶対に聞かない**）
   - 区分ごとに必要な書類（画像・PDF）をアップロード
4. アップロード直後に Claude API で自動抽出、申請者は抽出結果を確認・修正
5. 送信ボタンで最終送信
6. 管理者が内容を確認し、不備があれば差戻し

## 必要書類

### 法人

- 履歴事項全部証明書（発行から3ヶ月以内）
- 納税証明書その1（**法人税のみ可、消費税は不可**）
- 決算書（P/L・販管費・B/S の3点セット）

### 個人事業主

- 本人確認書類（運転免許証・運転経歴証明書・住民票）
- 確定申告書 第一表（受付印 or 電子申告受信通知 必須）
- 青色申告決算書（または白色の収支内訳書）
- 納税証明書その2（所得税）

## 検証ロジック

- 納税証明書の種類・税目を必ず検査（その1=法人税、その2=所得税）
- 履歴事項全部証明書の発行日は 90日以内
- 免許証は表裏両面必須 + 有効期限が切れていないこと
- 住民票は 90日以内
- 確定申告書は受付印 or e-Tax 情報が必要、なければ受信通知を追加提出
- 書類間で氏名/会社名/代表者名/住所が一致しているか（不一致は warning）

## セキュリティ要件

- GビズIDのパスワード入力欄を作らない
- localStorage に個人情報を保存しない
- 書類は Storage バケット内で 署名付きURL（時間制限あり）のみでアクセス
- ANTHROPIC_API_KEY はサーバーサイドのみ、クライアント露出厳禁
- 各書類のファイルサイズ上限 20MB
- 申請URLは30日有効期限付きのトークン
